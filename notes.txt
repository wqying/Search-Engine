Planning doc for my own use.

Inverted Index:
- data structure in IR systems
- maps words to their occurences within a set of docs
- fast querying, no need to scan entire document to look for words

Data Collection:
- foundation of building robust inverted index
- in our case, web scraping was used
- also the zip file provided by the professor

Data Cleaning:
- tokenize
- do not use stopping!
- use stemming (Porter stemming)
- must deal with bolded words, headings, titles as more important
- can get rid of punctuation though; lowercasing

Deliverables:
- number of indexed documents
- number of unique tokens
- total size of index on disk
